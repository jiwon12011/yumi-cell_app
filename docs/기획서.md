## 0. 한눈에 보기

> **"오늘의 나는 어떤 세포였을까?"**
> 매일 하루의 기분을 유미의 세포들 스타일의 세포 캐릭터 하나로 기록하는, 하루 한 줄 감정일기.

- 기록은 **세포 선택 + 강도(1~3) + 한 줄 노트(80자)** — 30초 안에 끝나는 초저마찰 기록
- 기록하면 그 세포가 **말풍선 멘트로 리액션** (날짜 기반 결정적 선택 — 같은 날 같은 멘트)
- **달력**에서 한 달의 감정 흐름을 세포 얼굴들로 조망, **통계**로 분포·연속 기록 확인
- **공유 카드**: 카드 프레임 + 세포 + 노트를 1080×1080 이미지로 생성해 공유/저장
- 프론트엔드 온리 · 로그인/서버 없음 · localStorage · 오프라인 동작(PWA)

| 스펙 | 값 |
|---|---|
| 플랫폼 | 모바일 웹 우선 (max-width 430px 중앙 컨테이너), 데스크톱 호환 |
| 스택 | Vite 8 + React 19 + TypeScript 6 + vite-plugin-pwa + Vitest 4 |
| 라우팅 | 라우터 없음. 뷰 상태 + 해시 딥링크(`#/d/YYYY-MM-DD`) |
| 저장 | localStorage 단일 키 `cell-mood-diary:v1` |
| 언어 | 한국어 |

## 1. 개요

### 1.1 배경
- 기존 기획(밸런스게임형 심리테스트)은 1회성 소비 콘텐츠라 재방문 동기가 약했다.
- 보유 에셋(세포 캐릭터 7종 × 포즈 5종, 카드 프레임, 파스텔 배경, 3D 아이콘/소품)은 **매일 얼굴을 보게 되는 일기·기록형 앱**에서 활용도가 가장 높다고 판단.
- 따라서 "하루 1회, 가볍게, 귀엽게"를 축으로 한 **감정 기록 앱**으로 전환한다.

### 1.2 목표
1. 하루 30초 안에 끝나는 기록 경험 → **매일 돌아오는 습관 루프** (스트릭·달력 수집욕)
2. 세포 캐릭터의 **리액션이 보상**이 되는 구조 (기록 = 세포가 대신 말해주는 내 기분)
3. 인스타 스토리 등에 올리고 싶은 **퀄리티의 공유 카드** → 자연 유입

### 1.3 타깃
- 유미의 세포들 감성을 좋아하는 10~30대
- 일기 앱은 부담스럽지만 "오늘 기분 정도는 남기고 싶은" 라이트 기록러

### 1.4 하지 않는 것 (Non-goals)
- 로그인·계정·서버 저장·랭킹 · 푸시 알림(v1) · 장문 일기 · 감정 분석 AI

## 2. 세포 캐릭터 정의

7종 전원이 동일한 파스텔 3D 치비 스타일(우주복 후드 + 흰 장갑/부츠 + 머리 안테나).

| id | 이름 | 대표색 | 성격 한 줄 | 말투 | 안테나/시그니처 |
|---|---|---|---|---|---|
| `emotion` | 감성세포 | `#FF6B9D` | 느낌이 먼저. 눈물도 감동도 많다 | "왜인지 몰라도~", "그냥 좋잖아" | 하트 안테나, 눈물, 하트 안기 |
| `reason` | 이성세포 | `#4C7DFF` | 계산·분석·결론부터 | "잠깐, 계산해보면", "결론은" | 동그란 안테나, 안경, 차트 태블릿 |
| `love` | 사랑세포 | `#FF4D6D` | 과몰입 로맨티스트 | "보고싶다", "걔는 지금 어떨까" | 하트 안테나, 하트 풍선/큰 하트 |
| `pride` | 자존심세포 | `#9B5DE5` | 지기 싫음, 쿨한 척 | "내가 먼저? 그건 좀", "아무렇지 않아(거짓말)" | 빨간 얼굴 + 뿔 + 정장, 팔짱 |
| `rest` | 휴식세포 | `#2EC4B6` | 나른·느긋·눕는 게 최고 | "이따 하면 되잖아", "누워있고 싶다" | 새싹 안테나, 선글라스, 베개/하품 |
| `anxiety` | 불안세포 | `#B79CED` | 걱정이 많고 조심스러움 | "혹시 잘못 고른 거 아냐?" | 땀방울, 체크리스트 뒤에 숨기 |
| `impulse` | 충동세포 | `#FF8A65` | 일단 지르고 본다 | "그냥 해버려!" | 번개 안테나, 확성기, 빨간 버튼 |

- `emotion`~`rest` 5종은 감정 기록의 메인 축, `anxiety`·`impulse` 2종도 **동등한 선택지** (구 기획의 scored 구분 폐지 — 일기에서는 모든 감정이 1등석).
- UI에서 세포별 색은 CSS 변수로 노출: `--cell-emotion: #FF6B9D` … `--cell-impulse: #FF8A65`.

## 3. 핵심 사용자 플로우

```
[첫 실행]                [매일]
온보딩 3장 ─▶ 오늘 탭 ─▶ 세포 선택 ─▶ 강도 선택 ─▶ (포즈 변경) ─▶ 한 줄 노트 ─▶ 저장
                                                                        │
              달력 탭 ◀── 홈으로 ──┐                                     ▼
              │  └ 날짜 탭 → 상세/수정/삭제                        리액션 화면
              통계 탭                                            (세포+말풍선 멘트)
                                                                  │
                                                        공유 카드 모달 → 공유/저장
```

- 오늘 이미 기록한 날 오늘 탭 진입 시 → **리액션 화면**을 먼저 보여주고 "수정하기" 버튼 제공.
- 과거 날짜는 달력에서 진입해 소급 기록 가능(단, 미래 날짜는 불가).
- 해시 딥링크 `#/d/2026-07-01` → 해당 날짜 상세(기록 있으면 보기, 없으면 작성 폼).

## 4. 화면 명세

### 4.0 공통 셸
- 최대폭 430px 중앙 정렬, 배경 밖 영역은 연핑크(`#FFF7FB`) 그라디언트.
- 하단 고정 **탭바 3개**: 오늘 / 달력 / 통계 (활성 탭은 세포 핑크 계열 강조, 44px 이상 터치 타깃).
- 아이콘은 `public/icons-v2/` 3D 아이콘 사용 (home, share, download, back, close, restart).
- 모든 화면 상단 safe-area 패딩(`env(safe-area-inset-top)`), PWA standalone 대응.

### 4.1 온보딩 (첫 실행 1회)
- 조건: 저장소에 entries가 하나도 없고 `onboarded` 플래그 없음.
- 슬라이드 3장 (스와이프/버튼, 건너뛰기 상시 노출):
  1. `backgrounds-v2/splash-room` 배경 + emotion 히어로 — **"오늘의 나는 어떤 세포일까?"**
  2. 세포 7종 얼굴 그리드 — **"매일 기분을 세포 하나로 기록해요"**
  3. 달력 목업 + crown 소품 — **"한 달 뒤, 내 감정의 지도가 생겨요"** → [시작하기]
- 완료 시 `onboarded: true` 저장 후 오늘 탭으로.

### 4.2 오늘 (기록 폼) — 기본 탭
- 배경: `backgrounds-v2/question-room` (하단 1/3은 흰 곡선 패널 영역 — 폼이 올라앉음).
- 헤더: `2026.07.02 수요일` + "오늘의 세포를 골라줘" (Jua체).
- **① 세포 선택 그리드**: 7종을 2행(4+3)으로, 각 셀 = pose-01 썸네일(원형 72px) + 이름. 선택 시 해당 세포 색 3px 링 + scale(1.06) + 살짝 bounce.
- **② 강도 칩 3개**: `살짝` `보통` `완전` (기본 `보통`). 선택 시 세포 색 채움. 강도에 따라 미리보기 포즈 자동 변경(§7.3).
- **③ 포즈 스와치 (접이식 "다른 포즈 보기")**: 해당 세포 01~05 썸네일 가로 스크롤, 탭하면 미리보기 교체(강도 기본값 override).
- **④ 미리보기**: 선택된 포즈 이미지(높이 ~180px)가 폼 상단에 크게 표시.
- **⑤ 한 줄 노트**: 1줄 input, `maxLength=80`, placeholder "오늘 한 줄… (선택)", 우측에 `n/80` 카운터. 빈 노트 허용.
- **[기록하기]** 버튼: 세포 미선택 시 비활성 + "세포를 골라줘!" 헬퍼 텍스트. 저장 → 리액션 화면 전환.
- 엣지: 오늘 기록 존재 시 이 화면은 "수정 모드"(기존 값 프리필, 버튼명 "수정하기").

### 4.3 리액션 (기록 직후 / 오늘 기록 열람)
- 배경: `backgrounds-v2/reaction-stage` (무대 + 커튼 — 세포가 주인공으로 서는 그림).
- 중앙: 선택 세포의 **히어로 이미지**(`cells-v2/{id}.png`, 높이 ~44vh) — bounce-in 등장(§9).
- 히어로 위 **말풍선(CSS로 그림, §8.4)**: `pickComment(cellId, intensity, date)` 멘트(§5). pop-in 애니메이션.
- 강도 3(완전)일 때 `decor-v2/confetti-dots` + `plus-sparkle` 오버레이가 잔잔히 떠다님.
- 하단 카드: 날짜 · 세포 이름(세포 색) · 내 노트 인용.
- 버튼: **[공유 카드 만들기]**(§4.6 모달) / [수정] / [홈] (아이콘: share, restart, home).
- 멘트는 날짜+세포+강도로 결정되므로 **재방문·새로고침에도 동일** — "오늘의 대사"로 인지되게.

### 4.4 달력
- 배경: 이미지 대신 **파스텔 CSS 그라디언트**(연보라→연핑크) — 썸네일 가독성·성능 우선.
- 헤더: `◀ 2026년 7월 ▶` (이번 달 이후로 이동 불가), 오늘로 점프 버튼.
- 월 그리드 7×6: 기록일 = 그 날의 `individual/{cellId}-{pose}` 썸네일(40px 원형, `loading="lazy"`), 미기록일 = 연회색 점, 오늘 = 세포 색 링, 미래 = 흐리게+탭 불가.
- 날짜 탭 → **바텀시트**: 포즈 이미지 + 강도 + 노트 + 멘트 / [수정] [삭제] [공유]. 미기록 과거 날짜면 그 날짜로 작성 폼.
- 삭제는 confirm 다이얼로그("이 날의 기록을 지울까요?") 후 실행.
- 월 요약 한 줄: "이번 달 12일 기록 · 감성세포가 5번으로 1등".

### 4.5 통계
- 이번 달(달력과 월 상태 공유) 기준:
  - **세포 분포**: 세포별 가로 바(세포 색, 좌측에 pose-01 미니 아이콘 28px, 우측 n회·%). 0회 세포는 흐리게.
  - **이번 달의 주인공**: 최다 세포 히어로 이미지(작게) + 카피 "이번 달은 {세포}의 달!". 동률이면 최근 기록 우선.
  - **스트릭**: `props-v2/crown` + "N일 연속 기록 중 / 최장 M일".
  - 기록 0건이면 빈 상태 일러스트(rest 세포 + "아직 기록이 없어… 오늘부터 시작!").

### 4.6 공유 카드 모달
- 1080×1080 canvas 미리보기(화면에는 축소 표시) + 하단 프레임 선택 스와치.
- **프레임 6종** = `cards-v2/backgrounds/` 전부. 세포별 기본 프레임:

  | 세포 | 기본 프레임 | 근거(무드) |
  |---|---|---|
  | emotion | pink-blue-cloud | 몽글 구름 |
  | reason | sky-cloud | 하늘색 |
  | love | cozy-desk | 하트 문구 데스크 |
  | pride | meeting-board | 보드/차트(성과 자랑) |
  | rest | night-dream | 밤/달 |
  | anxiety | lavender-panel | 라벤더 |
  | impulse | meeting-board | 스피치버블(외침) |

- 캔버스 레이아웃(1080 기준): 프레임 전면 → 포즈 이미지(중앙, y 180~700) → 하단 라운드 패널(x60 y740 w960 h280, 흰색 88% 불투명, r=40) → 노트(최대 2줄, `measureText` 줄바꿈, 48px) → 날짜+세포 이름(세포 색, 36px) → 우하단 워터마크 "세포 감정일기" 24px.
- 폰트: **Jua**(woff2 셀프호스팅, `public/fonts/`) — 캔버스 렌더 전 `document.fonts.load()` 대기, 실패 시 시스템 스택 폴백.
- [공유하기]: `canvas.toBlob(png)` → `navigator.canShare({files})` 지원 시 Web Share, 아니면 `<a download="세포일기-YYYY-MM-DD.png">` 자동 다운로드. 사용자 취소(AbortError)는 무시.

### 4.7 설정 (통계 탭 우상단 기어 → 시트)
- 항목: 데이터 초기화(2단 확인), 데이터 내보내기(JSON 다운로드), 앱 정보/버전.
- v1 범위는 이 3개만. 알림·백업 동기화는 오픈 이슈(§14).

## 5. 멘트 시스템

### 5.1 규칙
- 멘트 풀: **세포 7 × 강도 3 × 5줄 = 105줄** (아래 전체 카피 확정본).
- 선택: `index = hash(date + cellId + intensity) % 5` — 결정적(같은 입력 = 같은 멘트), 랜덤 금지.
- 길이 ≤ 40자(말풍선 2줄 이내), 이모지 미사용(데코는 이미지가 담당), 세포 말투 유지.

## 6. 데이터 모델 & 저장 정책

### 6.1 스키마
```ts
type CellId = 'emotion'|'reason'|'love'|'pride'|'rest'|'anxiety'|'impulse';

interface DiaryEntry {
  date: string;          // 'YYYY-MM-DD' — 로컬 타임존 기준. 키와 동일
  cellId: CellId;
  poseIndex: 1|2|3|4|5;
  intensity: 1|2|3;      // 1 살짝 / 2 보통 / 3 완전
  note: string;          // trim 후 최대 80자 (서로게이트 안전 카운트)
  createdAt: number;     // epoch ms
  updatedAt: number;
}

interface StoreV1 {
  version: 1;
  onboarded: boolean;
  entries: Record<string /* date */, DiaryEntry>;
}
```

### 6.2 정책
- localStorage 키: **`cell-mood-diary:v1`** 단일 키. entries map이 유일한 진실 원천 — 달력/통계/스트릭은 전부 순수 함수로 파생.
- **날짜 키는 로컬 타임존으로 생성** (`toISOString()` 금지 — UTC 밀림 버그 방지). 자정 넘김은 저장 시점 재계산.
- 복구 정책: JSON 파싱 실패 → 빈 스토어로 시작(크래시 금지). 항목 단위 검증(알 수 없는 cellId, 범위 밖 pose/intensity, 80자 초과)에 걸린 항목만 드롭/클램프.
- `version` 필드로 향후 마이그레이션 훅 확보 (v2 도입 시 v1→v2 변환 함수).
- 용량: 1년 365건 × ~200B ≈ 73KB — localStorage 한도(5MB) 대비 여유.
- 내보내기: 스토어 JSON을 그대로 `세포일기-백업-YYYY-MM-DD.json`으로 다운로드.

### 6.3 순수 함수 (전부 Vitest 유닛 테스트 대상)
| 함수 | 책임 | 핵심 테스트 |
|---|---|---|
| `todayISO(now)` | 로컬 날짜 문자열 | 타임존 경계(23:59/00:00) |
| `monthGrid(ym)` | 7×6 달력 셀 배열 | 윤년 2월, 12→1월 경계, 패딩 |
| `monthDistribution(entries, ym)` | 세포별 count/% | 합 100 보정, 빈 달 |
| `currentStreak(entries, today)` | 현재 연속일 | 오늘 미기록 시 어제부터, 월 경계 |
| `longestStreak(entries)` | 최장 연속일 | 공백 구간 |
| `pickComment(cellId, intensity, date)` | 멘트 결정 선택 | 결정성, 105조합 전부 멘트 존재 |
| `clampNote(s)` | trim+80자 | 서로게이트 페어(이모지) 안전 |
| `serialize/deserialize` | 스토어 IO | 라운드트립, 깨진 JSON, 불량 항목 드롭 |

- 원칙: 의미 있는 순간에만, 300ms 내외, CSS keyframes/transition만 사용(GSAP 등 외부 모션 라이브러리 도입 금지 — 규모 대비 과함).

| 대상 | 모션 | 값 |
|---|---|---|
| 리액션 세포 등장 | bounce-in (scale .6→1.05→1 + fade) | 450ms, cubic-bezier(.34,1.56,.64,1) |
| 말풍선 | pop (scale .8→1 + fade), 세포 등장 후 150ms 지연 | 250ms |
| 세포 선택 | 링 + scale 1.06 | 150ms |
| 탭 전환 | 콘텐츠 fade/slide 12px | 200ms |
| 강도3 파티클 | 위아래 float 무한 | 3s, 은은하게 |

- **`prefers-reduced-motion: reduce` 시 전 애니메이션 무효화**(즉시 최종 상태), 파티클 미표시.

## 10. 성능 계획

### 10.1 문제
원본 PNG 총 ~44MB, 배경 1장 1.1~1.9MB — 모바일 4G에서 배경 하나에 2~3초. 원본 직사용 불가.

### 10.2 이미지 파이프라인 (필수)
- `scripts/optimize-images.mjs` (devDependency **sharp**) → `public/optimized/`에 WebP 산출, **산출물을 커밋**(CI 불필요):

  | 입력 | 출력 | 예상 |
  |---|---|---|
  | backgrounds-v2 | 1080w q78 webp | 장당 ~120–250KB (≈85%↓) |
  | cells-v2 히어로 | 720w webp | ~80–150KB |
  | individual | **2종**: thumb 128w / mid 512w | 10–20KB / 40–80KB |
  | cards-v2/backgrounds | 1080w webp | ~200–300KB |
  | icons·decor·props | 160w webp | 수 KB |
  | mature-backgrounds | **파이프라인 입력에서 제외** | — |

- `data/assets.ts`가 모든 경로를 `/optimized/...`로 반환(이미지 경로의 단일 지점). WebP 폴백 불필요(지원율 사실상 100%).

### 10.3 로딩 정책
- 초기 로드: 현재 탭 배경 1장 + pose-01 thumb 7장 ≈ **250KB 이하** 목표. 첫 배경 `<img fetchpriority="high">`(absolute + object-fit: cover — CSS background 대신 img로 우선순위 제어).
- 예측 프리로드: 세포 선택 순간 → 그 세포 히어로 + reaction-stage 배경 워밍(리액션 전환 시 캐시 히트).
- 달력 썸네일: `loading="lazy" decoding="async"` + width/height 고정(CLS 0).
- 공유 모달: 기본 프레임 1장만 즉시, 나머지 5장은 스와치 노출 시 로드.

### 10.4 PWA (vite-plugin-pwa)
- manifest: name `세포 감정일기`, short_name `세포일기`, theme `#FF6B9D`, bg `#FFF7FB`, display standalone, lang ko. 아이콘 192/512(+maskable)는 sharp 스크립트에서 emotion 캐릭터 기반 생성.
- precache: 앱 셸(js/css/html/폰트) + optimized icons/decor만. **대형 이미지(배경·히어로·카드)는 precache 금지** → `runtimeCaching: CacheFirst, cacheName 'images', maxEntries 120, 30일`.
- 성능 예산: 3G Fast 기준 첫 인터랙션 3초 이내, JS 번들(gzip) ≤ 90KB, Lighthouse 모바일 Performance 90+.

## 11. 접근성
- 모든 이미지 버튼에 `aria-label`(예: "공유하기"), 세포 그리드는 `role="radiogroup"`/`radio` + 키보드 화살표 탐색.
- 강도 칩·포즈 스와치도 radio 패턴. 포커스 링 항상 표시(세포 색 outline).
- 색만으로 정보 전달 금지(분포 바에 이름+수치 병기). 터치 타깃 44px+. 바텀시트 열릴 때 포커스 이동/트랩, 닫으면 원위치.
