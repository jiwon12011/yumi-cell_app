## 1. 화면 구성 (5화면)

모바일 세로 우선(360~480px), `safe-area-inset` 고려.

### 01 Splash (1~2초)
- 중앙 로고/세포, 파스텔 그래디언트 배경. 세포들이 회의실로 입장하는 무드.
- 탭하면 즉시 skip.

### 02 Intro
- 타이틀 + 태그라인 + 설명 + "회의 참석하기" 버튼.
- 세포 미리보기 5개(72px) 가볍게 등장.

### 03 Question (7회 반복)
- 상단: 진행도 `Q n / 7` + 프로그레스 바(높이 6px).
- 상황 지문(H3) + **A/B 선택 카드 2개**(최소 높이 60px, 터치 44px+).
- 선택 후 "다음" 버튼 노출.
- 질문 카드 배경에 그 질문에서 크게 반응할 세포 컬러를 **은은한 틴트**로(채도 낮게).

### 04 Reaction (선택 후)
- 선택된 맥락의 세포 캐릭터 + **말풍선 2~3개가 스태거드 리빌**(0.15s 간격 톡톡).
- 말풍선 다 뜬 뒤 "다음" 버튼 활성(연타 방지). **자동 진행 아님** — 대사 읽는 맛이 핵심.

### 05 Result
- 대표 세포 등장(글로우) + 결과 제목/해석 카드 + 점수 바(선택) + **다시하기 / 공유** 버튼.
- 오늘의 세포 운세 한 줄.
- 공유 카드(세포 대사 한 줄 포함) 저장/공유.

### 3-2. 질문 7개

> 형식: 상황 / A·B 라벨 / 점수(맵) / 세포 말풍선(누가)

**Q1** — 주말 아침. 알람 없이 눈이 떠졌다. 오늘 아무 계획 없음.
- **A** "이불 속에서 좀 더 뒹굴기" — `{H:2, E:1}`
  - 휴식: "이게 맞지. 역시 나야." / 감성: "오늘 이 기분 너무 좋아" / 충동: "일단 유튜브만 켜봐"
- **B** "오래 못 본 친구한테 연락하기" — `{L:2, E:1}`
  - 사랑: "맞아 보고싶다 진짜로" / 감성: "왜 갑자기 설레지?" / 불안: "갑자기 연락하면 이상한가"

**Q2** — 점심 메뉴를 골라야 한다. 혼자 먹을 거라 내 맘대로 가능.
- **A** "어제부터 먹고 싶었던 거 그냥 먹는다" — `{E:2}`
  - 감성: "이게 내 오늘이야" / 사랑: "맞아 이거 나도 좋아해" / 충동: "대자로 시켜 후회 없이"
- **B** "영양소랑 칼로리 따져서 결정한다" — `{R:2}`
  - 이성: "그러니까 이쪽이 맞잖아" / 자존심: "이 정도는 챙겨야지" / 불안: "근데 맛있긴 한 건가?"

**Q3** — 카톡 보냈는데 읽음 표시 뜨고 답장이 없다.
- **A** "내가 뭔가 잘못했나... 계속 생각한다" — `{E:2}`
  - 감성: "내가 뭘 잘못한 걸까..." / 사랑: "걔 요즘 무슨 일 있나" / 불안: "설마 나 싫어진 거 아니지?"
- **B** "나도 바빠서 그럴 때 있지, 넘긴다" — `{H:2, R:1}`
  - 휴식: "어 나도 그럴 때 있어" / 이성: "모르면 그냥 기다려" / 충동: "한 번 더 보내볼까?"

**Q4** — 길 가다가 전 연인이랑 딱 마주쳤다.
- **A** "못 본 척하고 지나간다" — `{P:2}`
  - 자존심: "이게 맞아. 내가 먼저?" / 이성: "어색한 것보다 낫잖아" / 충동: "아 한 번 말 걸어볼까?"
- **B** "\"어! 안녕, 잘 지냈어?\" 먼저 인사한다" — `{L:2, E:1}`
  - 사랑: "잘 지냈길 바랐어 진짜" / 감성: "왜 이렇게 두근거리지" / 자존심: "...쿨하게 잘했어 그래도"

**Q5** — 인스타에 사진 올리려는데.
- **A** "각도, 밝기, 필터 다 맞추고 올린다" — `{P:2}`
  - 자존심: "이 정도는 해야지 당연" / 감성: "이게 지금 내 느낌이야" / 불안: "뭔가 좀 이상하지 않아?"
- **B** "기억용 사진이라 그냥 바로 올린다" — `{H:2, R:1}`
  - 휴식: "기억이면 충분해 그냥" / 이성: "과정 생략하면 효율이야" / 충동: "그냥 올려버려 빠르게"

**Q6** — 내가 한 말에 친구가 반박해왔다. "그건 좀 아닌 것 같은데."
- **A** "\"그럴 수도 있겠다, 생각해볼게\"" — `{R:2, L:1}`
  - 이성: "그러니까 한번 생각해봐" / 사랑: "관계가 더 중요하잖아" / 자존심: "...억울하긴 한데"
- **B** "\"아니 근데 내 말이 맞는데?\"" — `{P:2}`

**Q7** — 갑자기 마음에 드는 사람이 생겼다.
- **A** "설레는 감정 그냥 느끼면서 기다려본다" — `{L:2, E:1}`
  - 사랑: "이 감정 더 느껴봐" / 감성: "설레는 게 좋잖아 원래" / 불안: "근데 상대방은 어떨까?"

### 3-4. 오늘의 세포 운세 (세포별 10개, 날짜해시 `yyyymmdd % 10`)

**감성**: ①느낌이 맞으면 그게 답이야 ②감동받을 일이 생길 것 같은데 ③뭔가 설레는 하루가 될 거야 ④좋아하는 음악 들어. 기분 달라져 ⑤울고 싶으면 울어. 오늘은 괜찮아 ⑥직감 믿어봐. 맞아 ⑦작은 것도 크게 느끼는 날이야 ⑧보고 싶은 감정, 오늘은 무시하지 마 ⑨감성이 올라오는 날. 그냥 흘러가 ⑩오늘 하루 뭔가 소중하게 느껴질 거야

**이성**: ①생각 정리하고 결정해 ②감정보다 사실을 먼저 봐 ③분석하면 답이 나와 ④복잡한 상황도 쪼개면 간단해져 ⑤머리가 잘 돌아가는 날 ⑥잠깐 계산해봐. 맞을 수도 ⑦감정 흔들릴 때 논리로 잡아 ⑧오늘 내린 결론 믿어봐 ⑨효율적인 하루가 될 거야 ⑩정리하면 홀가분해질 거야

**사랑**: ①생각났으면 연락해봐 ②먼저 손 내밀면 좋은 날 ③따뜻한 말 한마디 해봐 ④그 사람 지금 뭐 하고 있을까 ⑤보고 싶은 사람 오늘 만나 ⑥표현 안 하면 아무도 몰라 ⑦내 마음 조금만 더 솔직하게 ⑧사랑은 먼저 하는 사람이 이겨 ⑨연락 와도 안 와도 오늘은 괜찮아 ⑩누군가에게 웃어줘봐

**자존심**: ①아무렇지 않은 척 잘하고 있어 ②선 지켜도 돼. 네 기준이 맞아 ③먼저 연락 안 해도 되는 날 ④쿨한 척이 진짜 쿨함이 되는 날 ⑤네 의견 틀리지 않았어 ⑥양보 안 해도 돼. 버텨 ⑦지켜야 할 선이 있다면 지켜 ⑧자존심은 약점이 아니야 ⑨오늘 지지 않아도 돼 ⑩인정받고 싶은 마음, 이상한 거 아니야

**휴식**: ①이거 하나만 하고 쉬어 ②쉬는 것도 계획이야 ③억지로 하면 안 되는 날 ④충전이 먼저야. 다 나중에 ⑤아무것도 안 해도 돼 ⑥이불이 답인 날, 오늘이야 ⑦무리하지 마. 내일이 있어 ⑧쉬어도 된다는 허락, 내가 줄게 ⑨한 템포 줄여봐 ⑩게으른 게 아니야, 충전 중인 거야

### 카피 톤 규칙
- 세포 대사 ≤ 20자, 반말·구어, 세포별 말투 구분.
- 결과/운세 "~했어/~야" 톤. **"~입니다/유형입니다" 진단서 말투 금지.**

## 4. 기술 구조 (developer)

- **스택**: Vite + React + TS, `vite-plugin-pwa`. 라우팅 라이브러리 없음(결과만 `?r=` 딥링크). 상태관리 `useReducer` 하나. 데이터는 `src/data/*.ts` 정적 모듈(오프라인 동작).
- **관통 원칙(척추)**: 상태는 `history`(선택 기록 배열) **하나만 진실**. 점수·진행도·결과는 전부 `derive(history)` 순수함수로 파생.
  - 뒤로가기 = `history.pop()`, 다시하기 = `history = []`. **점수 되돌리기 로직 없음 → 버그 원천 차단.**

### 데이터 모델
```ts
type CellId = 'emotion' | 'reason' | 'love' | 'pride' | 'rest'; // 점수 5종
// 분위기 2종(불안/충동)은 reaction.cell 로만 쓰이고 scored:false

interface Cell   { id; name; color; accent; tone; scored: boolean }
interface CellReaction { cell: string; text: string }
interface Choice { key:'A'|'B'; label; reactions: CellReaction[]; scores: Partial<Record<CellId,number>> }
interface Question { id; index; situation; choices: [Choice, Choice] }
interface ResultType { id: CellId; cell; title; subtitle; description; cardImage; shareText; cellQuote }
```

### 점수 집계 & 동점 처리 (결정론, 랜덤 금지)
1. **최고점** 세포가 단독이면 그게 결과.
2. 동점 → **최근성 가중**: `history` 역순으로 가장 늦게 점수받은 세포. (마지막 질문 Q7 → Q6 순으로 가중)
3. 그래도 동점 → 고정 우선순위 배열. (안전망)

### 상태머신 (useReducer)
- `phase: splash | intro | question | reaction | result`
- 액션: `START / CHOOSE(key) / NEXT / BACK / RESTART`
- 파생값: `currentIndex = history.length`, `progress`, `scores = tally(history)`, `result = deriveResult(history)`
- 딥링크 `?r=emotion` 진입 → 결과 화면 바로 부팅.

### 밸런스 단위테스트 (코드 전 안전장치)
- results는 5 CellId를 1:1 커버 / 모든 scores·reaction.cell 이 유효 id / 5세포 각각 최다 가능 경로 존재. → 빌드/테스트가 콘텐츠 깨짐을 즉시 잡음.

## 6. 모션 (motion-engineer)

- 이징 2종: **톡** `cubic-bezier(.34,1.56,.64,1)` / **스르르** `cubic-bezier(.22,1,.36,1)`.
- 시그니처: **말풍선 스태거드 리빌** — delay `0.15s + i*0.15s`, duration `0.38s`, 시작 `translateY16px scale .82 opacity0`, **톡**. transform/opacity만.
- 카드 선택: 선택 카드 `scale 1→1.05→1`(톡), 미선택 `opacity→.42 scale→.97`. 점수 바 `scaleX`(width 금지).
- 구현: **GSAP 미사용** (CSS keyframe/transition + 인덱스 주입). `--i` custom property로 delay 계산.
- **reduced-motion**: 모든 translate/scale 즉시, opacity 페이드만, stagger 제거, 글로우 정지. 내용 노출 동일.
- 60fps 금지: width/height·top/left·border-radius 애니메이션 금지, will-change 남발 금지, backdrop-filter 주의.

## 8. 공유 & 부가기능

- **결과 카드 공유**: 결과가 5종 고정이므로 **정적 카드 5장**을 미리 제작(`/public/cards/{cellId}.png`). html2canvas 런타임 생성 회피.
- **Web Share API 3단 폴백**: ①파일첨부 공유 → ②텍스트+URL 공유 → ③클립보드 복사 + 이미지 다운로드.
- 결과 딥링크 `?r={cellId}`.
- **오늘의 세포 운세 한 줄**(날짜해시 랜덤, 백엔드 X) — 재방문 유도.
- 질문 카드 세포컬러 틴트(채도 낮게).
- *v2 보류*: 친구 결과 비교(URL 인코딩).

## 9. 성능 게이트 (perf-engineer) — ✅ 조건부 통과

반영 필수 3조건:
1. **말풍선 backdrop-filter 금지** → `rgba(...,0.92)` + 노이즈 SVG 질감. 글래스는 결과카드 히어로 1장만.
2. **Pretendard 한글 서브셋 100KB↓** (woff2, `font-display:swap`, fallback 메트릭 매칭).
3. **결과카드 5장 PWA precache 분리** — 설치 시 precache 금지, 마지막 답 선택 시 해당 1장 프리로드, 나머지 4장 prefetch.

용량 예산: 초기 JS(gzip) 80KB↓ / CSS 15KB↓ / 폰트 100KB↓ / LCP까지 200KB↓ / 결과카드 각 240KB↓(합계 1.2MB↓).
LCP 후보: Splash/Intro 히어로 세포 1종(`fetchpriority=high` preload).
CLS: 모든 이미지 width/height 예약, 말풍선 공간 사전 예약. INP: 답변→다음 질문 패스 16ms↓.
