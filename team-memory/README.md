# 우리팀 브레인 (team-memory)

우리팀이 *여러 프로젝트를 거치며 학습한 것*을 누적하는 자리. 이 폴더는 `jiwon-harness` 본체에 산다 — 즉 모든 프로젝트가 공유하는 단 하나의 뇌다.

원칙: **우리팀은 매 프로젝트에서 원점에서 시작하지 않는다.** 과거에 통한 패턴·결정·도메인 지식을 읽고 시작한다.

## 구조

```
team-memory/
├── lessons/    프로젝트별 회고 시트 (templates/lesson.md 형식)
├── decisions/  팀 차원 의사결정 (ADR 형식, 한 결정 한 파일)
├── patterns/   여러 프로젝트에서 통한 재사용 패턴
└── domain/     특정 영역에서 얻은 지식 (포트폴리오·랜딩·앱 등)
```

## 새 프로젝트 시작할 때 (team-lead가 반드시)

팀 모드로 의미 있는 작업이 시작되면, **team-lead가 시작 정리 직전에 이 폴더를 한 번 훑는다** — "이 프로젝트와 관련된 과거 결정·패턴·도메인 지식이 있나?" 확인하고, 있으면 시작 정리에 한 줄로 끌어온다.

> team-lead 도구는 Read/Glob/Grep만 가지므로 이 폴더를 읽을 수 있다. 플러그인으로 설치된 경우 경로는 플러그인 설치 위치 하위 `team-memory/`이고, 레포에 직접 설치된 경우 `./team-memory/`다.

## 학습을 누적하는 법 (성장 의례)

자동으로는 안 자란다 — *명시적 회고*로 자란다.

1. 프로젝트(또는 큰 작업 한 덩어리)가 끝나면 `/retro` 또는 "회고하자"로 회고를 연다.
2. team-lead가 `templates/lesson.md`를 채운다 (팀원별 한 줄 + Keep/Drop/Try/Domain).
3. *본체로 옮길 항목*에 체크된 것만 → 이 `jiwon-harness` 레포에 **PR**.
4. 머지되면 다른 기기에서 플러그인을 갱신(`/plugin` 업데이트)하거나 레포를 `git pull` → 학습된 우리팀이 다음 프로젝트로 따라온다.

> 이 의례를 빼먹으면 우리팀은 다음 프로젝트에서 원점이다. team-lead가 큰 작업 끝에 환기한다.
