---

이 명령은 우리팀의 **성장 의례**를 실행한다 — 이번 작업에서 배운 것을 본체(`jiwon-harness`)로 옮겨 다음 프로젝트가 원점에서 시작하지 않게 한다.

다음을 수행하라:

1. **team-lead 호출** (Agent 도구, subagent_type: `team-lead`) — 이번 작업/프로젝트의 회고를 주도하게 한다.
   - `templates/lesson.md` 형식으로 회고를 채운다: Keep / Drop / Try / Domain + 팀원별 한 줄.
   - 팀원별 한 줄은 실제로 그 작업에 관여한 팀원을 Agent로 호출해 받는다(관여 안 한 팀원은 생략 가능). 메인이 대신 적지 않는다.
2. **회고 시트 저장** — 작업 중인 레포 안에 남기려면 `./.jiwon-team/retro-<날짜>-<슬러그>.md`로 저장한다(없으면 폴더 생성). CEO가 본체에만 남기길 원하면 이 단계는 건너뛴다.
3. **본체로 옮길 항목 정리** — 회고의 "본체로 옮길 항목"에서 체크된 것만 추린다. 각 항목마다 *어느 파일에 / 어떤 룰로* 들어갈지 한 줄로 명시한다(`team-memory/patterns/…`, `team-memory/domain/…`, 또는 `agents/<팀원>.md`).
4. **PR 제안** — 옮길 항목이 있으면, `jiwon-harness` 레포에 올릴 변경안(어떤 파일에 무엇을 추가/수정)을 사용자에게 보여주고 승인받는다. 승인되면:
   - `gh`가 있고 인증돼 있으면 `learn/<프로젝트>-<날짜>-<슬러그>` 브랜치로 PR을 만든다.
   - 없으면 변경 내용을 마크다운으로 출력해 사용자가 직접 본체에 반영하도록 안내한다.
   - **자동 머지하지 않는다** — 본체에 박는 건 사용자(서지원)가 PR에서 직접 결정한다.
5. 옮길 게 없으면 "이번엔 본체로 승격할 학습 없음 — 회고만 남김"으로 한 줄 보고하고 끝낸다.

> 핵심: 회고는 team-lead가, 팀원 코멘트는 각 팀원 에이전트가 직접 쓴다. 메인 클로드가 팀원 시늉으로 채우지 않는다.
