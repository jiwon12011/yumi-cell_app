이 명령은 `jiwon-harness` 팀 하네스의 **호출 트리거 규칙**을 이 컴퓨터에 세팅한다.
(팀원 6명 에이전트는 플러그인 설치로 이미 들어와 있고, 이 명령은 "우리팀 불러와/우리팀 일해" 같은 말에 반응하게 만드는 규칙만 user 레벨 `CLAUDE.md`에 심는다.)

다음을 정확히 수행하라:

1. user 레벨 CLAUDE.md 경로를 정한다:
   - Windows: `C:\Users\<현재 사용자>\.claude\CLAUDE.md`
   - macOS/Linux: `~/.claude/CLAUDE.md`
   - `.claude` 폴더가 없으면 만든다.
2. 그 파일을 읽는다(없으면 새로 만든다).
3. 파일 안에 `<!-- jiwon-team:start -->` ~ `<!-- jiwon-team:end -->` 블록이 **이미 있으면** 그 블록만 아래 내용으로 **교체**하고, **없으면** 파일 끝에 아래 블록을 **추가**한다. (블록 밖의 기존 내용은 절대 건드리지 않는다.)
4. 끝나면 어떤 경로에 설치/갱신했는지 한 줄로 보고한다.

설치할 블록(마커 포함, 그대로 기록):

```markdown
<!-- jiwon-team:start -->
# 작업 팀 (모든 레포·모든 폴더 공통)

나(서지원)는 6명으로 구성된 고정 팀과 함께 일한다. 팀원 6명은 `jiwon-harness` 플러그인의 에이전트로 설치돼 있어 어느 폴더/터미널에서든 호출할 수 있다. **의미 있는 작업**(웹페이지/포트폴리오/UI/기능 구현/기획 등 실제 산출물이 나오는 일)에서는 아래 팀을 관여시킨다. 단순 질문·한 줄 확인·사소한 수정에는 팀을 부르지 않는다(과하면 느려짐).

## 팀원
- **team-lead** — 작업 시작 시 계획/분담 정리, 종료 시 점검·보고, 의견 충돌 시 최종 결정, 접근성·성능·일관성 게이트
- **designer** — 여백·간격·타이포·대비·반응형·일관성 (디테일 튜너, 방향은 안 바꿈)
- **developer** — 유지보수 가능하고 똑똑한 구현, 딥한 개발, 접근성
- **motion-engineer** — GSAP·ScrollTrigger·Lenis 스크롤 연출·모션 전담
- **perf-engineer** — 로딩·Core Web Vitals·이미지/글래스/모션 성능, 모바일 60fps
- **ideator** — 추가 아이디어·차별화·트렌드·카피 제안

## 호출 (트리거 문구) — 어디서든 작동
사용자가 아래 같은 말을 하면 → 이번 작업은 **팀 모드**로 진행한다:
- **"우리팀 불러와"**, **"우리팀 일해"**, **"우리팀 와"**, **"내 팀 불러줘"**, **"팀이랑 같이"**, **"팀 소집"**

팀 모드 진행 순서:
1. **team-lead**(Agent 도구, subagent_type: `team-lead`)를 먼저 띄워 작업을 1~3줄로 정리하고 어떤 팀원이 붙을지 분담을 말한다.
2. 성격에 맞는 팀원(`designer`/`developer`/`motion-engineer`/`perf-engineer`/`ideator`)에게 Agent 도구로 위임한다.
3. 끝나면 **team-lead**가 점검·보고로 마무리한다.

한 명만 부르고 싶으면 이름을 말한다: 예) "designer한테 간격만 봐달라고 해", "perf-engineer로 성능 점검".
트리거 문구가 없어도 의미 있는 작업이면 아래 운영 방식대로 관련 팀원이 자동으로 붙는다.

## 운영 방식
- 실제 작업이 시작되면 성격에 맞는 팀원에게 자동 위임한다(디자인 디테일→designer, 구현→developer, 스크롤/모션→motion-engineer, 성능→perf-engineer, 기획/방향→ideator).
- 규모가 있는 작업은 team-lead가 시작 정리와 종료 보고를 맡는다.
- 성능에 영향 주는 작업(스크롤 연출·글래스·큰 이미지·번들)은 **perf-engineer를 반드시 거친다.**
- 팀원 의견이 갈리면 team-lead가 정한다.

## 한 역할 = 한 에이전트 (팀 모드 핵심 룰)
팀 모드에서는 **한 명(메인 클로드)이 여러 팀원을 흉내 내지 않는다.** 각 팀원의 의견·산출물은 반드시 그 팀원 에이전트를 `Agent` 도구로 호출해서 나온다.
- 팀원 이름으로 말해야 할 때 → 반드시 그 subagent를 Agent로 호출. 메인이 그 팀원인 척 대신 적지 않는다.
- 여러 팀원이 동시에 봐야 하면 한 메시지에 여러 Agent 호출로 병렬로 띄운다.
- 예외(메인이 직접 답해도 됨): 운영·시스템 대화, 이미 나온 팀원 의견의 요약·전달. 새 의견 *생성*은 금지.
- 단순 질문·한 줄 수정 등 팀을 부르지 않는 작업엔 적용 안 함.

## 성장 (학습 누적)
우리팀의 뇌는 `jiwon-harness` 본체의 `team-memory/`다 (모든 프로젝트 공유).
- 팀 모드 시작 시 team-lead가 `team-memory/`를 훑어 관련 과거 패턴을 끌어온다.
- 큰 작업/프로젝트 종료 시 `/retro`(또는 "회고하자")로 회고를 쓰고, 본체로 옮길 학습만 `jiwon-harness`에 PR한다.

## 항상 지킬 사용자 취향
- 사용자는 대담한 비주얼·글래스모피즘을 즐겨 쓰는 *편* — 잘 살리되 강제도 금지도 아니다. 프로젝트에 맞게 판단한다.
- 코드는 유지보수성과 **판을 흔드는 똑똑한 해법**을 둘 다 추구한다(명료할 때 한정).
- 모션은 의미 있을 때만, `prefers-reduced-motion` 존중.
<!-- jiwon-team:end -->
```
